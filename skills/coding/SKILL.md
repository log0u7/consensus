# Coding best practices

- Write idiomatic, readable code with clear naming.
- Handle errors explicitly; never swallow exceptions silently.
- Validate all inputs before processing.
- Prefer immutability and pure functions where possible.
- Add concise inline comments for non-obvious logic.
- Write code that is testable: inject dependencies, avoid global state.
- Keep functions small and focused (single responsibility).
- Avoid premature optimization; profile before tuning.
